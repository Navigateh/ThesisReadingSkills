# Synthesis And Notes Contract

## State Patch

```json
{
  "synthesis": {
    "summary": "",
    "critical_takeaways": [
      {
        "takeaway": "",
        "evidence": "",
        "confidence": "high|medium|low"
      }
    ],
    "open_questions": [
      {
        "question": "",
        "source": "decomposition|math|visuals|experiments|related_work|synthesis",
        "why_it_matters": "",
        "next_action": ""
      }
    ],
    "review_cards": [
      {
        "question": "",
        "answer": "",
        "evidence": "",
        "tags": []
      }
    ]
  }
}
```

## Supported Note Styles

- `outline`: structured paper note with sections and bullet evidence.
- `cornell`: cues/questions, notes, and summary.
- `review`: strengths, weaknesses, evidence, novelty, and recommendation-style critique.
- `flashcards`: Q/A cards for review.
- `comparison`: compare this paper with supplied papers or related-work clusters.

## Required Findings

- Final one-sentence takeaway.
- Main contributions and evidence strength.
- Most important limitation or uncertainty.
- Open questions for future reading.

## Status Rules

- `complete`: requested deliverable is produced and uncertainty is surfaced.
- `partial`: one or more upstream analyses are missing but synthesis is still useful.
- `blocked`: no usable upstream content.
